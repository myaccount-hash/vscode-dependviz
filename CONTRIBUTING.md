# コミットメッセージ規約

形式は `type: summary` に統一する．

## type

- feat: 機能追加
- fix: バグ修正
- docs: ドキュメントのみ
- refactor: 振る舞いを変えない整理
- chore: ビルド・設定・依存関係
- test: テストのみ

## summary

- 1行で簡潔に書く
- 末尾にピリオドを付けない

## 例

- feat: add isearch exit on any key
- fix: avoid crash on empty selection
- docs: add hotkey config example
